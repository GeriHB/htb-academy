# Obfuscation

## Code Obfuscation

A technique to make a script more difficult to be read by humans.

Usually done automatically using an obfuscation tool.

Usually it turns the code into a dictionary of all of the words and symbols used within the code and rebuilds the original code during execution by referring to each word and symbol from the dictionary.

Usually is done to hide the original code and its functions from being reused or copied without permission.

Another reason is to prevent attacks on vulnerabilities that can be found within the code.

But most common usage is for malicious actions.

## Basic Obfuscation

Let's take the following JS code and obfuscate it.

```js
console.log('Obfuscation');
```

**Minifying JS Code**
It's a common way to reduce the readibility while keeping it functional. 

This means that the entire code will be in one simple line.

In our case it doesn't do much help, because the whole code is one line, but it works quite well for long pieces of code.

**Packing JS code**
If we use the online tool [Beautifytools](https://beautifytools.com/javascript-obfuscator.php) the simple code we had became:

```js
eval(function(p,a,c,k,e,d){e=function(c){return c};if(!''.replace(/^/,String)){while(c--){d[c]=k[c]||c}k=[function(e){return d[e]}];e=function(){return'\\w+'};c=1};while(c--){if(k[c]){p=p.replace(new RegExp('\\b'+e(c)+'\\b','g'),k[c])}}return p}('0.1(\'2\');',3,3,'console|log|Obfuscation'.split('|'),0,{}))
```

This is a `packer` obfuscation tool which converts all words and symbols of the code into a list and then refer to them using the `(p, a, c, k, e, d)` function to re-build the original code during the execution.

It usually has a certain order in which the words and symbols of the original code were packed to know how to order them during the execution.

The downside is taht we can still see its main strings written in cleartext.

**Reverse Engineering**
When the code is heavy obfuscated and encoded, it is difficult to deobfuscate it it with automated tools.

Especially when the obfuscation is done using custom tools, not the public ones.

And to deobfuscate them we need to manually reverse engineer the code and understand how it was obfuscated.
